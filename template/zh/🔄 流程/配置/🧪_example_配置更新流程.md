# ⚙️ Agent MCP/Skill 一次配置 SOP 示例

## 目标

- 把 Agent 的 MCP 与 Skill 配置固化到 MindOS。
- 避免每次新会话都重复手动配置。

## 触发条件

- 新增或更换 MCP 服务。
- 新增或升级常用 Skill。
- Agent 行为协议有变更（读取顺序、执行边界、安全规则等）。

## 前置输入

- 目标配置清单（MCP、Skill、行为规则）。
- 当前模板路径：`template/zh/`。

## 步骤

1. 新建或更新 Agent 协议文档
   - 在目标业务目录下按需创建文档（例如 `🤖 Agent 全局协议.md`）。
   - 明确读取顺序、规则优先级、执行边界与安全限制。

2. 新建或更新 MCP 清单文档
   - 记录 MCP 名称、用途、接入方式与最小验证步骤。
   - 不写入明文密钥，仅写获取与注入流程。

3. 新建或更新 Skill 清单文档
   - 记录 Skill 名称、触发条件、安装方式与使用边界。

4. 同步配置协议文件
   - 若涉及命名/层级/生成行为调整，同步更新 `CONFIG.json` 与 `CONFIG.md`。

5. 运行一致性检查
   - 用 `rg` 检查旧路径、旧命名、失效引用。
   - 确认 `INSTRUCTION.md` 与 `README.md` 描述一致。

## 验收标准

- Agent 在新会话中可直接按文档加载 MCP/Skill，无需临时补充口头配置。
- 配置文档与目录结构一致，无失效路径。
- 中英文模板语义保持一致（若本次改动影响 en 侧，需同步更新）。

## 复盘建议

- 记录本次配置后节省的重复操作（例如：无需再次手配 MCP/Skill）。
- 若仍出现重复手配，补充缺失字段到相应目录的配置文档。
