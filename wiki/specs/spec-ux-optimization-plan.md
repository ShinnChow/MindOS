# MindOS 产品体验优化计划

*创建时间：2026-04-01 | 来源：全面 UX 审计 + 产品辩证分析*

---

## 诊断

**整体评分：7/10 — 功能齐全，指引不足。**

问题不在"缺什么"，而在"指路不清"：
- 核心功能都有了（文件 + AI Chat + Agents + Echo + Export + Trash）
- 用户**找不到**核心功能（"Echo" 名字不明白、"Discover" 被忽视）
- 空状态后又循环回新手引导，形成心理陷阱
- Agent 连接需要开终端 + 复制粘贴，破坏了 Web UX

---

## P0（<1 天）— 改变第一印象

### 1. 重命名模糊面板

| 当前 | 改为 | 理由 |
|------|------|------|
| Echo | 洞察 / Insights | "Echo" 不自解释，用户不会点 |
| Discover | 探索 / Browse | "Discover" 对非英语用户模糊 |
| Flows | 自动化 / Automations | "Flows" 太抽象 |

**为什么**：名字不自解释 = 功能不存在。这是 ROI 最高的改动。

### 2. 空状态从"循环"变"检查清单"

当前：选完模板 → 首页空 → 再看新手引导（"这是什么？"）

改为：选完模板 → 显示 3 步检查清单：
```
□ 写下你的第一条想法
□ 用 ⌘/ 和 AI 聊聊
□ 连接你的第一个 AI 工具（ChatGPT / Cursor）
```
每完成一个 ✅ 消失。满足感强，目标明确。

### 3. CSV / 文件操作添加保存反馈

当前：点保存 → 无反馈 → 用户不知道是否成功。

改为：Toast "已保存" / "保存失败，请重试"。一行代码，防止无效编辑。

### 4. Agent 连接：减少终端依赖

当前：Settings → Agents → MCP → 复制 token → 打开终端 → 粘贴（5 步 + 离开浏览器）

改为：Agent 面板内 → 选择 Agent → "一键配置"按钮 → 自动写入配置（或生成二维码/链接）

## P1（1 周）— 高频操作体验升级

### 5. Ask AI 上下文透明化

当前：⌘/ 打开 Ask → 用户不知道 AI 看的是当前文件还是全部笔记。

改为：Ask 面板顶部显示 "在与 {文件名} 交流" 或 "与全部笔记交流"（可切换）。

### 6. 搜索结果加上下文片段

当前：⌘K 搜索 → 只显示文件名 → 要点开才知道内容。

改为：搜索结果显示 "...{搜索词} 前后 10 字..." 片段，带高亮。

### 7. 文件头部加 Ask 按钮

当前：看完一个文件 → 想聊 → 要记住内容，然后 ⌘/ 再打字。

改为：文件顶栏右侧有 "Ask AI" 按钮 → 直接打开 Ask + 自动填入当前文件名。

### 8. AI 响应阶段可见化

当前：Ask AI → "Connecting..." → 等 → 回复（用户不知道在干嘛）。

改为：分阶段进度：
```
🔍 搜索相关笔记...
📖 阅读上下文...
🤔 思考中...
✍️ 生成回复...
```

## P2（2-4 周）— 产品定位澄清

### 9. 首页故事化

当前首页太稀疏，不说明 MindOS 的价值。

改为：
```
┌─────────────────────────────────────┐
│  MindOS：你和 AI 共享的大脑          │
│                                     │
│  📝 记录想法 → 🤖 AI 理解           │
│  💡 洞察自动生成 → 🎯 更聪明地工作   │
│                                     │
│  ┌─────────┬──────────┬──────────┐  │
│  │  1 条笔记│ 0 个 AI  │ 0 条洞察  │  │
│  └─────────┴──────────┴──────────┘  │
│                                     │
│  [ 新建笔记 ] [ 连接 AI ]          │
└─────────────────────────────────────┘
```

### 10. 行为驱动的功能发现（替代双模式）

不做"标准/专业"二选一切换，而是用行为触发渐进功能：
- 默认显示全部 Activity Bar，新功能用 Badge 标记
- 10+ 次 Agent 交互 → 推荐 "Workflows"
- 100+ 文件 → 推荐创建 "Space"
- 纠正 AI 一次错误 → 提示保存为 Skill

核心原则：**不是"你是谁"决定看到什么，而是"你做了什么"决定看到什么。**

## 衡量成功

| 指标 | 当前（估） | 目标 |
|------|-----------|------|
| 首次用户"发现 Echo/Insights"率 | ~20% | >60% |
| 首次用户"连接 Agent"成功率 | ~30% | >80% |
| 平均首次使用时长 | ~5 min | >15 min |
| "这个产品是什么"清晰度 | 5/10 | 8/10 |

---

## 与 Progressive Disclosure Spec 的关系

本计划是 `spec-progressive-disclosure.md` 的**务实落地版**：
- JIT 概念引入 → 保留（P2.10 行为驱动发现）
- 空状态双语气 → 简化为统一检查清单（P0.2）
- 双模式切换 → 推迟到有用户数据后决策
- 面板重命名 → 立即执行（P0.1）

**优先做能用数据验证的小改动，再决定大架构。**
